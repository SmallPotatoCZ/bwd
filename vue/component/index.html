组件注册

组件名

两种方式：

  1. kebab-case - <kebab-case>
  2. PascalCase - <PascalCase> 或者 <pascal-case>，直接在 DOM 中只有 kebab-case。

全局注册

子组件直接可以互相使用。

Vue.component("cmpt-name", {/*...*/});

局部注册

// 在其子组件中不可用。
var ComponentA = {/*...*/};
var ComponentB = {/*...*/};
new Vue({
    el:"#app",
    components:{
        "cmpt-a": ComponentA,
        "cmpt-b": ComponentB
    }
})

// 如果要互相使用
var ComponentA = {/*...*/};
var ComponentB = {
    components: {
        'component-a': ComponentA
    }
};

模块系统

模块系统中局部注册

import ComponentA form './ComponentA'

export default {
    components: {
        ComponentA
    }
}

基础组件的自动化全局注册

const requireComponent = require.context(
  // 其组件目录的相对路径
  './components',
  // 是否查询其子目录
  false,
  // 匹配基础组件文件名的正则表达式
  /Base[A-Z]\w+\.(vue|js)$/
)

requireComponent.keys().forEach(fileName => {
  // 获取组件配置
  const componentConfig = requireComponent(fileName)

  // 获取组件的 PascalCase 命名
  const componentName = upperFirst(
    camelCase(
      // 获取和目录深度无关的文件名
      fileName
        .split('/')
        .pop()
        .replace(/\.\w+$/, '')
    )
  )

  // 全局注册组件
  Vue.component(
    componentName,
    // 如果这个组件选项是通过 `export default` 导出的，
    // 那么就会优先使用 `.default`，
    // 否则回退到使用模块的根。
    componentConfig.default || componentConfig
  )
})

传递静态或动态 Prop

<!-- 字符串 -->
<blog-post title="My journey with Vue"></blog-post>
<!-- 动态赋值一个复杂的表达式 -->
<blog-post v-bind:title="post.title + 'by' + post.author.name"></blog-post>

传入数字

<!-- 通过 v-bind 标识 42 是表达式 -->
<blog-post v-bind:likes="42"></blog-post>

<!-- 通过变量动态赋值 -->
<blog-post v-bind:likes="post.likes"></blog-post>

传入布尔值

<!-- prop 没有值的情况表示 true -->
<blog-post is-published></blog-post>

<!-- v-bind 表明 false 是表达式 -->
<blog-post v-bind:is-published="false"></blog-post>

<!-- 通过变量动态赋值 -->
<blog-post v-bind:is-published="post.isPublished"></blog-post>

传入数组

<!-- v-bind 表明静态的数组是表达式 -->
<blog-post v-bind:comment-ids="[234,266,273]"></blog-post>

<!-- 变量动态赋值 -->
<blog-post v-bind:comment-ids="post.commentIds"></blog-post>

传入对象

<!-- v-bind 表明静态的对象是表达式 -->
<blog-post v-bind:author="{
  name: 'zc',
  company: 'Veridian Dynamics'
}"></blog-post>

<!-- 用动态变量进行动态赋值 -->
<blog-post v-bind:author="post.author"></blog-post>

传入一个对象的所有属性

post:{
  id:1,
  title:'My Journey with Vue'
}

<blog-post v-bind="post"></blog-post>
<!-- 等价于 -->
<blog-post v-bind:id="post.id" v-bind:title="post.title"></blog-post>

单项数据流

父级的 prop 的更新会向下流动到子组件，反过来不可以。防止子组件意外的改变父组件的状态。

试图改变 prop 值的两种情况:

1.prop 传递初始值，接下来希望作为一个本地的数据使用。

  props: ["initialCounter"],
  data: function(){
    return {
      counter: this.initialCounter
    }
  }

2.prop 是原始的值且需要转换

  props: ['size'],
  computed: {
    normalizedSize: function(){
      return this.size.trim().toLowerCase()
    }
  }

数组或对象类的 prop ,在子组件中改变这个对象或者数组本身将会影响到父组件。

Prop 验证

Vue.component("my-cmpt",{
  props:{
    // 基础类型检查（`null`和`undefined` 会通过任何类型的验证）
    propA: Number,
    // 多个类型
    propB:[String,Number],
    // 必填字段
    propC:{
      type:String,
      required:true
    }
    // 带有默认的数字
    propD:{
      type:Number,
      default:100
    },
    // 带有默认值的对象
    propE:{
      type:Object,
      default:function(){
        return{message:'hello'}
      }
    },
    // 自定义验证函数
    propF:{
      validator:function(value){
        // 这个值必须匹配下列字符串中的一个
        return ['success','warning','danger'].indexOf(value) !== -1;
      }
    }
  }
})

会在组件创建之前进行验证，所以实例的属性（如：data、computed等）在default或validator函数中不可用的。
